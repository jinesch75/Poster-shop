admin: add QR re-stamp action to maintenance backfill page

The /admin/maintenance/backfill page previously only handled the
one-time migration of seeded `public:` masters onto the Railway volume.
After the rebrand to Gridline Cities (with the new domain
gridlinecities.com), every existing poster's preview still carries a QR
code that encodes the old Railway URL — but the existing backfill skips
any volume-backed poster, so it can't be reused for this purpose.

This commit adds a second server action `runRestamp` and a second card
on the same page. The new action walks every volume-backed poster,
calls reprocessMaster() on its existing master to regenerate previews +
thumbnails + mockups, and writes the new derivative keys back to the
DB. The QR badge encodes qrTargetUrl(slug), which reads from
NEXT_PUBLIC_SITE_URL — so with that env var now set to
https://gridlinecities.com, re-stamped posters get fresh QR codes
pointing at the new domain.

The original migration action stays untouched in case any future poster
ever lands on the `public:` prefix again.

Page layout now has two cards:

  Migrate legacy public: masters    [Run migration on N posters]
  Re-stamp QR codes                  [Re-stamp N posters]

After this deploys, hit /admin/maintenance/backfill on
https://gridlinecities.com and click "Re-stamp 21 posters". The pipeline
should produce fresh previews encoding gridlinecities.com/q/<slug> for
every London + New York poster.

The page <h1> is now "Poster image maintenance" rather than "Backfill
legacy masters" since it now serves both purposes; the route URL stays
/admin/maintenance/backfill so existing bookmarks still work.
